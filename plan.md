## 1. 現況概述

### 平台簡介

**EduGenius AI 智學** 是專為香港 DSE 學生打造的 AI 輔助學習平台，現有 6 大科目、逾 24 種 AI 工具，採用 Pollinations AI 作為推理引擎，Firebase 作為認證與資料庫。

### 現有功能一覽

| 頁面 | 功能 |
|------|------|
| `index.html` | 主頁、登入、科目選擇、Ask AI / Dictionary AI / Guide Learning、歷史紀錄面板 |
| `coding.html` | ICT 程式碼審查平台（Python） |
| `exam.html` | DSE 考試卷生成器（數學、物理、生物） |
| `vocab.html` | 英語詞彙生成器 + 詞彙測驗 |
| `eng-writing.html` | 英語寫作練習 |
| `mensyu2.html` | 文樞：古典文學互動學習 |
| `mensyu-tran.html` | 文言文翻譯工具 |

### 現有不足（按嚴重程度）

| 問題 | 類別 | 嚴重程度 |
|------|------|---------|
| API 金鑰暴露於前端 | 安全 | 🔴 嚴重 |
| 無串流輸出，等待體驗差 | UX | 🟠 高 |
| 無學習進度追蹤 | 教學 | 🟠 高 |
| 各頁面設計不一致 | 介面 | 🟡 中 |
| 詞彙測驗無間距重複（SRS）| 教學 | 🟡 中 |
| ICT 只支援 Python | 功能 | 🟡 中 |
| 深色模式僅 index.html 支援 | 介面 | 🟡 中 |
| 無考試卷品質管控 | 教學 | 🟡 中 |
| 歷史紀錄無搜尋功能 | UX | 🟢 低 |

## 2. 教學提升計劃

### 2.1 強化 AI 教學模式

#### 2.1.1 新增「蘇格拉底＋解構式」引導學習升級版

**現況：** Guide Learning 僅問引導問題，學生容易卡關。

**計劃：**
- 在 Guide Learning 加入「提示梯度」（Hint Ladder）：學生可選擇「再給我一個提示」逐步獲得更直接的線索，而非直接給答案。
- 每個回應結尾加入「自我反思問題」（Reflection Prompt），如「你認為這個方法有什麼局限？」
- 若學生連續答錯 3 次，系統自動切換至「部分解釋」模式並提示轉用 Ask AI。

#### 2.1.2 新增「錯誤分析」模式（Error Analysis Mode）

**現況：** 無針對學生答錯的分析功能。

**計劃：**
- 在每個 AI 工具加入「我的答案」輸入框，學生可貼上自己的答案。
- AI 識別錯誤類型（概念錯誤、計算錯誤、語言表達錯誤等），並針對性解說。
- 生成「個人化錯誤報告」，儲存至 Firestore，讓學生追蹤常犯錯誤。

#### 2.1.3 系統提示（System Prompts）優化

**現況：** 各科系統提示質素不一，部分過於簡短。

**計劃：** 針對每科 DSE 課程重新撰寫更細緻的系統提示：

**中文：**
- 區分指定篇章（12 篇）與自由篇章
- 納入 DSE 中文卷一、二、三、四各卷答題技巧
- 加入文言文修辭手法識別指引

**英語：**
- 分 Paper 1（閱讀）、Paper 2（寫作）、Paper 3（聆聽）、Paper 4（口語）指引
- 加入 IELTS 式評分反饋（Task Achievement、Coherence、Lexical Resource、Grammar）
- 內置常見語法錯誤糾正模板（冠詞、時態、句子結構）

**數學（必修 + 延伸）：**
- 區分 Module 1（統計與微積分）與 Module 2（代數與微積分）
- LaTeX 公式渲染強化，確保每個解題步驟清晰
- 加入「解題策略選擇」提示（如「這題應用 AM-GM 不等式還是柯西不等式？」）

**物理：**
- 按 DSE 考綱 Topics 1-12 分類回答
- 實驗題（Experiment Questions）加入標準格式回應模板
- 加入單位換算提醒與有效數字規範

**生物：**
- 強調 DSE 評分標準用語（accepted terms）
- 圖表分析題（Graph Questions）加入結構化分析框架
- 遺傳學題目加入遺傳圖（Punnett Square）ASCII 表示

**ICT：**
- 加入程式碼偵錯（Debugging）專項提示
- 資料庫查詢（SQL）輔助解說
- 網絡概念（OSI 模型、TCP/IP）圖解說明

### 2.2 詞彙學習提升（vocab.html）

#### 2.2.1 間距重複系統（Spaced Repetition System, SRS）

**現況：** 詞彙測驗為靜態隨機出題，無學習效率優化。

**計劃：**
- 實作 SuperMemo SM-2 演算法：根據答對/答錯調整下次複習時間間隔
- 以 `localStorage` 儲存每個詞彙的熟悉度評分（0-5）、複習間隔、下次複習日期
- 加入「今日待複習」提醒（Daily Review Queue）

#### 2.2.2 詞彙難度分級

**計劃：**
- 將詞彙按 DSE 難度分為 3 級：基礎（Foundation）、進階（Advanced）、卓越（Excellence）
- 測驗前允許學生選擇難度
- 視覺化熟悉度進度條（每個詞彙顯示熟練程度 0-100%）

#### 2.2.3 多元題型

**現況：** 測驗僅有填充題。

**計劃：**
- 選擇題（Multiple Choice）
- 句子填充（Sentence Completion）
- 配對題（Matching，詞彙↔定義）
- 情境造句（Contextual Sentence Writing，AI 評分）

### 2.3 英語寫作提升（eng-writing.html）

#### 2.3.1 實時寫作評分

**計劃：**
- 學生提交作文後，AI 按 DSE 評分準則給出：
  - **內容分**（Content）：論點是否切題、論據是否充分
  - **語言分**（Language）：文法、詞彙、句式多樣性
  - **組織分**（Organisation）：段落結構、連接詞使用
- 分項顯示評分（1-7 分制，對應 DSE Level 1-5+）

#### 2.3.2 寫作模板庫

**計劃：**
- 提供 DSE 常見文體模板：
  - 議論文（Argumentative Essay）
  - 電郵/信件（Email/Letter）
  - 新聞稿（Report）
  - 演講稿（Speech）
  - 故事寫作（Short Story）
- 每個模板顯示結構說明 + 示例段落

#### 2.3.3 詞彙豐富度建議

**計劃：**
- AI 分析學生作文，標記重複詞彙
- 為每個重複詞彙提供 3-5 個同義詞及例句
- 加入「用詞升級」（Word Upgrade）功能：一鍵替換平淡詞彙為更高分詞彙

### 2.4 考試卷生成器提升（exam.html）

#### 2.4.1 擴展至中文科

**現況：** 只支援數學、物理、生物。

**計劃：** 新增中文科考試卷生成，包含：
- 卷一：閱讀理解（選 DSE 真題格式）
- 卷二：寫作（提供不同情境題）
- 卷三：聆聽及綜合能力（提示練習格式）

#### 2.4.2 試卷品質管控

**現況：** AI 直接生成題目，無驗證機制。

**計劃：**
- 加入「二次驗證」步驟：AI 生成題目後，再次請 AI 驗算答案（Self-Verification Chain）
- 在試卷上顯示「AI 生成，請核對答案」免責聲明
- 允許學生回報問題（Report Error 按鈕）

#### 2.4.3 難度自適應

**計劃：**
- 學生選擇目標等級（Level 2 / 3 / 4 / 5 / 5* / 5**）
- 系統根據等級調整題目難度與評分標準
- 顯示「本試卷對應 DSE XX 年份難度」參考

### 2.5 ICT 程式碼審查提升（coding.html）

#### 2.5.1 多語言支援

**現況：** 只支援 Python（.py, .ipynb）。

**計劃：** 新增以下語言支援：
- **JavaScript / Node.js**（DSE ICT 電腦概論）
- **HTML/CSS**（網頁設計）
- **SQL**（資料庫）
- **Java**（延伸選修）

#### 2.5.2 程式碼逐行解說

**計劃：**
- 除現有「程式碼審查」外，新增「逐行解說」模式
- AI 在每個關鍵程式碼行旁加入繁體中文注釋
- 識別「學生程式碼常見反模式」（Anti-patterns）並建議重構

#### 2.5.3 互動式程式碼補全練習

**計劃：**
- 加入「填空程式碼」（Code Completion）題目
- AI 生成帶空白的程式碼，學生填寫後提交
- 系統評估填寫是否正確並解說最佳答案

### 2.6 文樞（mensyu2.html）與文言文翻譯（mensyu-tran.html）提升

#### 2.6.1 12 篇指定篇章深度整合

**計劃：**
- 為每篇指定篇章（如《廉頗藺相如列傳》《岳陽樓記》等）建立專屬學習頁面
- 包含：全文白話譯文、重要字詞解釋、修辭手法分析、DSE 考點提示
- 互動注釋：點擊文言字詞即顯示解釋（Tooltip 功能）

#### 2.6.2 作者背景與文學脈絡

**計劃：**
- 在文樞加入「作者檔案」功能，AI 介紹作者生平及其對文章風格的影響
- 加入「時代脈絡」說明（如唐宋、先秦等不同時期文風差異）

### 3.2 串流輸出（Streaming Responses）

**現況：** AI 回應需完整生成後才顯示，長回應等待時間可達 10-30 秒。

**計劃：**
- 改用 Pollinations AI 的串流 API（`stream: true`）
- 逐字元/逐 Token 顯示 AI 回應（仿 ChatGPT 打字效果）
- 串流過程中顯示「正在輸入...」指示器
- 允許學生中途停止生成（Stop 按鈕）

**技術實現（概要）：**
```javascript
// 現況（非串流）
const response = await fetch(API_URL, { body: JSON.stringify({...data, stream: false}) });
const result = await response.json();

// 計劃（串流）
const response = await fetch(API_URL, { body: JSON.stringify({...data, stream: true}) });
const reader = response.body.getReader();
// 逐 chunk 讀取並即時渲染
```

## 4. 介面設計優化計劃

### 4.1 設計系統統一化

**現況問題：**
- `exam.html` 使用純自訂 glassmorphism CSS，無 Tailwind
- `mensyu2.html` 使用藍綠色系，與主平台紫色系不符
- `vocab.html` / `eng-writing.html` 為完全自包含頁面，設計語言不一致
- 深色模式僅 `index.html` 支援

**計劃：** 建立統一設計規範文件（Design Tokens）

```css
/* 統一 CSS 變數（所有頁面共用） */
:root {
  /* 主色 */
  --primary: #7c3aed;
  --primary-light: #a78bfa;
  --primary-dark: #5b21b6;
  --secondary: #06b6d4;
  --secondary-light: #67e8f9;

  /* 科目標識色 */
  --subject-chinese: #ef4444;
  --subject-english: #3b82f6;
  --subject-math: #10b981;
  --subject-physics: #8b5cf6;
  --subject-biology: #22c55e;
  --subject-ict: #6366f1;

  /* 玻璃效果 */
  --glass-bg: rgba(255, 255, 255, 0.80);
  --glass-border: rgba(255, 255, 255, 0.60);
  --glass-blur: blur(24px) saturate(160%);

  /* 間距與圓角 */
  --radius: 20px;
  --radius-sm: 12px;
  --radius-xs: 8px;

  /* 字型 */
  --font-primary: 'Plus Jakarta Sans', 'Inter', sans-serif;
  --font-cjk: 'Microsoft JhengHei', 'PingFang TC', sans-serif;
  --font-code: 'Monaco', 'Menlo', 'Ubuntu Mono', monospace;

  /* 陰影 */
  --shadow-sm: 0 4px 16px rgba(0, 0, 0, 0.06);
  --shadow-md: 0 8px 32px rgba(0, 0, 0, 0.08);
  --shadow-lg: 0 20px 60px rgba(0, 0, 0, 0.12);
}
```

#### 4.1.1 深色模式全平台統一

**計劃：** 為所有頁面（coding.html、exam.html、vocab.html、eng-writing.html、mensyu2.html、mensyu-tran.html）加入深色模式支援：
- 使用 `localStorage` 持久化主題偏好（目前重整頁面後會重設）
- 主題切換時加入 CSS transition（防止閃爍）
- 深色模式下玻璃效果自動調整透明度

#### 4.3.1 科目卡片升級

**現況：** 科目卡片為靜態卡片，無互動回饋。

**計劃：**

## 5. 安全與後端改善

### 5.1 🔴 高優先：API 金鑰保護

**現況：** Pollinations AI API 金鑰直接暴露在前端 JavaScript 中。

**計劃：**
- 建立輕量後端代理（可用 Cloudflare Workers 或 Firebase Cloud Functions）
- 前端向代理發送請求，代理持有金鑰並轉發至 Pollinations AI
- 加入速率限制（Rate Limiting）：每用戶每分鐘最多 10 次請求
- 實作基於 Firebase Auth Token 的請求驗證

### 5.2 Firestore 安全規則加強

**現況：** 現有規則雖有基本保護，但部分驗證不夠嚴格。

**計劃：**
- 強制驗證 `tool`、`subject` 欄位只能為合法值（白名單）
- 加入每日訊息限制（每用戶每天最多 500 條訊息）
- 防止 `summary` 欄位注入惡意內容（長度限制已有，加入字元過濾）

### 5.3 錯誤監控

**計劃：**
- 整合 Firebase Crashlytics 或 Sentry（免費方案）監控前端錯誤
- 建立 AI API 請求失敗率追蹤（Fallback 模型使用統計）
- 學生遇到錯誤時可一鍵回報（帶錯誤快照）
